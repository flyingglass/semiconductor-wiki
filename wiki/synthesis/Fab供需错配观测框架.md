# Fab 供需错配观测框架

基于霍华德·马克斯泡沫分析框架，向上游推导至 Fab 环节的供需错配评估方法。

## 一、两层放大机制

```
终端用户付费（T0，真实需求）
    ↓ 第一层放大
互联网巨头 capex（T-2年 前置投入，半真半假）
    ↓ 第二层放大
芯片设计商向 Fab 下单（牛鞭效应）
    ↓
Fab 报营收增长 → 但分不清三种需求来源
```

Fab 营收混入了：真实终端需求 + 大厂 capex 前置 + 地方债务驱动需求。分离这三者是核心难题。

## 二、中美 Fab 链分列

### 美国链：台积电先进制程（3nm/5nm）

驱动源：英伟达 → CSP。错配表现为英伟达下单 > CSP 真实采购需求。

| 指标 | 当前（2026中） | 含义 |
|------|--------------|------|
| 英伟达 DC 营收增速 | 极高，边际放缓？ | 需求仍在，加速度变化 |
| 英伟达库存天数 | 低位 | 尚未堆积 |
| 台积电先进制程交期 | 26周+ | 产能紧张 |
| 利用率 | ~95%+ | 满负荷 |

**错配判断**：当前错配不大。CSP 真实采购追得上供货。至少 2027 年前看不到供给过剩。

**台积电明斯基时刻**：CSP 砍 GPU → 英伟达库存堆积 → 向台积电砍单 → 交期缩短 → 利用率下降。

### 中国链：中芯国际/华虹成熟制程（14nm/28nm+）

驱动源：华为昇腾/寒武纪 → 城投算力中心。错配表现为双重放大——芯片设计师下单 > 城投真实采购 > 终端付费。

| 指标 | 当前（2026中） | 含义 |
|------|--------------|------|
| 中芯国际成熟制程利用率 | ~85% | 尚未明显下降 |
| 成熟制程 ASP | 2026涨价10% | 仍在高位 |
| 寒武纪应收账款 | 需跟踪 | 攀升 = 城投拖账 |
| 中央审计通报 | **已发生** | 第一个官方确认信号 |

**错配判断**：错配在积累。审计信号刚发出。

**中芯国际明斯基时刻**：审计整改 → 城投砍单 → 寒武纪营收断崖 → 向中芯砍单 → 利用率 85%→60%。

## 三、四指标观测框架（从下游往上游）

按时序排列：

| T+N | 指标 | 观测对象 | 领先 Fab 多久 |
|-----|------|---------|-------------|
| T+0 | GPU 租赁价格下跌 | 第三方平台报价 | 12-18个月 |
| T+6 | 大厂 capex/revenue 比见顶 | 字节/阿里/腾讯财报 | 6-12个月 |
| T+9 | 芯片设计商库存天数上升 | 英伟达库存/寒武纪存货 | 2-3个季度 |
| T+12 | Fab 交货周期缩短 | 台积电/中芯国际交期 | 1-2个季度 |
| T+15 | Fab 营收增速放缓 | 财报 | 当下 |

### 当前时点位置（2026.06）

- GPU租赁价格——有下跌信号（待更可靠验证）
- 大厂 capex——字节仍在加码（2,000亿+）
- 芯片商库存——低位
- Fab 交期——紧张

→ **可能处于 T+0 到 T+3 位置。台积电营收拐点在 4-6 个季度后。**

### 共振信号

当 GPU 租赁价格和 Fab 交货周期**同时下行** → 全链条共振 → 明斯基时刻从下游传到上游。

## 四、润泽科技作为观测窗口

润泽是字节最大第三方 IDC 供应商（3万P+算力），位于两个关键交点：

### 可观测信号

| 观测点 | 数据来源 |
|--------|---------|
| 字节真实上架率 | 签约算力 vs 实际上架算力 |
| 字节租约结构 | 年限、保底条款、续约率 |
| GPU 来源 | 自采占 50%，剩余从哪转租（城投？） |
| 算力定价趋势 | 每P租金季度环比 |
| capex 前置程度 | 签约率 vs 实际上架率差 |

### 组合观测

- 润泽 + 数据港 → 字节需求全貌
- 利通电子 → 阿里/腾讯需求
- 三家合在一起 → 互联网大厂 capex 去向全图
- 交叉寒武纪应收账款 + 中芯国际利用率 → 中国链供需错配完整拼图
